# 0-1背包问题 — 回溯法

## 题目描述

有 n 个物品，每个物品有重量 w[i] 和价值 v[i]。给定一个容量为 c 的背包，每个物品只能选一次（0-1），求在不超过背包容量的前提下能装入的最大总价值。

**示例：**

> 输入: 重量 `w = [12, 5, 8, 9, 6]`，价值 `v = [9, 2, 4, 7, 8]`，容量 `c = 12`
> 输出: 最大价值 15（选重量 6+5 或 12 的物品）

## 思路分析

**子集树模型**：每个物品选或不选，共 $2^n$ 种组合。

**状态设计：**
- `cw` — 当前已选物品的总重量
- `cv` — 当前已选物品的总价值
- `r` — 剩余未处理物品的总价值（用于限界剪枝）

**剪枝策略（核心优化）：**

| 策略 | 条件 | 作用 |
|------|------|------|
| 可行性剪枝 | `cw + w[i] <= c` | 超重则不选 |
| **限界剪枝** | `cv + r > bestv` | 即使全选剩余物品也无法超过当前最优解，则剪枝 |

> 限界剪枝是最关键的优化，大幅减少搜索空间。

## C++ 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int w[] = {12, 5, 8, 9, 6};  // 重量
int v[] = {9, 2, 4, 7, 8};   // 价值
const int len = sizeof(w) / sizeof(w[0]);
int c = 12;                   // 背包容量

int cw = 0, cv = 0;           // 当前重量/价值
int bestv = 0;                // 最优价值
int r = 0;                    // 剩余物品总价值
vector<int> bestx, x;         // 最优解 / 当前解

void func(int i) {
    if (i == len) {            // 到达叶子节点
        if (cv > bestv) {
            bestv = cv;
            bestx = x;         // 记录最优选择
        }
        return;
    }
    
    r -= v[i];                 // 从剩余价值中减去当前物品
    
    // 尝试放入当前物品（可行性剪枝）
    if (cw + w[i] <= c) {
        cv += v[i];
        cw += w[i];
        x.push_back(w[i]);
        func(i + 1);
        cv -= v[i];
        cw -= w[i];
        x.pop_back();
    }
    
    // 限界剪枝：不放当前物品，但剩余价值足以超过最优解才搜索
    if (cv + r > bestv) {
        func(i + 1);
    }
    
    r += v[i];                 // 恢复剩余价值
}

int main() {
    for (int val : v) r += val;
    func(0);
    
    for (auto val : bestx)
        cout << val << " ";
    cout << endl;
    cout << "bestv:" << bestv << endl;
    return 0;
}
```

## 复杂度分析

| 维度 | 指标 |
|------|------|
| 时间复杂度 | $O(2^n)$（无剪枝），剪枝后平均远小于此值 |
| 空间复杂度 | $O(n)$，递归栈深度 |

## 易错点 / 拓展思考

- ❗ `r` 的更新时机：进入 `func(i)` 时就要 `r -= v[i]`，因为 i 已经被"处理"了
- ❗ 限界条件 `cv + r > bestv` 是 >= 还是 >？如果要求严格最优，`>` 即可
- ❗ 注意 `r` 在递归左子树和右子树之间的恢复顺序
- **拓展 1**：本题也可以用**动态规划**求解，空间复杂度 $O(c)$，时间复杂度 $O(n \cdot c)$
- **拓展 2**：如果每个物品可以选多次（完全背包），回溯框架如何调整？
