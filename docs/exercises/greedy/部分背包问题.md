# 部分背包问题

## 题目描述

有 n 个物品，每个物品有重量 $w_i$ 和价值 $v_i$。在总重量不超过容量 $c$ 的情况下让总价值尽量高。**每个物品可以只取走一部分**，价值和重量按比例计算，求最大价值。

**示例：**

> 输入: 重量 `w = [8, 6, 4, 2, 5]`，价值 `v = [6, 4, 7, 8, 6]`，容量 `c = 12`
> 输出: 最大价值 17.6

## 思路分析

**核心思想**：按**性价比**（价值/重量）降序排列，优先装性价比高的物品。

```
物品:        A     B     C     D     E
重量:        8     6     4     2     5
价值:        6     4     7     8     6
性价比:     0.75  0.67  1.75   4    1.2

按性价比排序: D(4) → E(1.2) → C(1.75) → A(0.75) → B(0.67)
放入过程:
  D: 重量2≤12 ✅ 放入，c=10
  E: 重量5≤10 ✅ 放入，c=5
  C: 重量4≤5  ✅ 放入，c=1
  A: 重量8>1  ⚠️ 按比例放入 1/8，价值 6×(1/8)=0.75
  总价值 = 8 + 6 + 7 + 0.75 = 21.75
```

## C++ 代码

```cpp
struct Product {
    double getPrice() const {
        return v * 1.0 / w;  // 性价比
    }
    bool operator>(const Product &p) const {
        return getPrice() > p.getPrice();
    }
    int id;  // 物品编号
    int w;   // 重量
    int v;   // 价值
};

int main() {
    int w[] = {8, 6, 4, 2, 5};
    int v[] = {6, 4, 7, 8, 6};
    int c = 12;
    int n = sizeof(w) / sizeof(w[0]);
    int x[n] = {0};   // 选中的物品
    
    Product pros[n];
    for (int i = 0; i < n; ++i) {
        pros[i] = {i, w[i], v[i]};
    }
    
    // 按性价比降序
    sort(pros, pros + n, [](const Product &p1, const Product &p2) {
        return p1 > p2;
    });
    
    double bestv = 0.0;
    for (int i = 0; i < n; ++i) {
        if (pros[i].w <= c) {
            bestv += pros[i].v;     // 整件装入
            c -= pros[i].w;
        } else {
            bestv += pros[i].v * (c * 1.0 / pros[i].w);  // 按比例装入
            x[pros[i].id] = 1;
            break;                   // 背包已满
        }
        x[pros[i].id] = 1;
    }
    
    cout << "bestv:" << bestv << endl;
    return 0;
}
```

## 复杂度分析

| 维度 | 指标 |
|------|------|
| 时间复杂度 | $O(n \log n)$，主要为排序 |
| 空间复杂度 | $O(n)$，存储物品结构体 |

## 易错点 / 拓展思考

- ❗ 排序降序需要用 `operator>`（或 lambda 中 `return p1.getPrice() > p2.getPrice()`）
- ❗ 按比例装入后要 `break`，背包已满不能再装后续物品
- ❗ 与 **0-1 背包** 不同：部分背包可以用贪心（物品可分割），0-1 背包需要 DP
- **拓展 1**：如果物品不可分割（0-1 背包），贪心还正确吗？为什么？
- **拓展 2**：如果物品可以取任意多次（完全背包），贪心是否还适用？
