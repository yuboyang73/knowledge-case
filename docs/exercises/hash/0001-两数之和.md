# 0001. 两数之和

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** 的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素不能使用两遍。

**示例 1：**

> 输入: `nums = [2,7,11,15], target = 9`
> 输出: `[0,1]`
> 解释: 因为 `nums[0] + nums[1] == 9`

**示例 2：**

> 输入: `nums = [3,2,4], target = 6`
> 输出: `[1,2]`

**示例 3：**

> 输入: `nums = [3,3], target = 6`
> 输出: `[0,1]`

## 思路分析

核心思想：**一边遍历，一边查哈希表**。

```
遍历数组 nums：
  对于当前元素 nums[i]，计算 complement = target - nums[i]
  在哈希表中查找 complement：
    ✅ 找到 → 返回 {complement的下标, i}
    ❌ 未找到 → 将 nums[i] 加入哈希表 {值 → 下标}
```

> 选用 unordered_map 而非 set，因为需要同时记录值和下标。

## C++ 代码

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> map;    // key: 数值, value: 下标
        for (int i = 0; i < nums.size(); i++) {
            auto it = map.find(target - nums[i]);
            if (it != map.end())
                return {i, it->second}; // 找到互补元素，返回下标
            map[nums[i]] = i;           // 记录当前元素
        }
        return {};                      // 题目保证有解
    }
};
```

## 复杂度分析

| 维度 | 指标 |
|------|------|
| 时间复杂度 | $O(n)$，只需遍历一次数组 |
| 空间复杂度 | $O(n)$，哈希表最多存储 n 个元素 |

## 易错点 / 拓展思考

- ❗ 不能先全部存入 map 再遍历，否则同一个元素会被用两次
- ❗ 题目要求返回下标，所以必须用 `unordered_map`，不能用 `unordered_set`
- ❗ 数组中可能存在相同值，但题目保证每个输入只有一个答案
- 拓展：如果数组是**有序**的，可以用左右双指针做到 $O(1)$ 额外空间
