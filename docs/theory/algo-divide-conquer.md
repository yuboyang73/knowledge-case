# 分治算法

## 基本思想

规模为 n 的原问题的解无法直接求出时：

1. **分解** — 划分子问题（子问题相互独立，性质与原问题相同，规模缩小）
2. **求解** — 递归求解子问题，直到规模足够小可直接求解
3. **合并** — 将子问题的解合并为原问题的解

```
        原问题 (规模 n)
        /    |    \
   子问题  子问题  子问题    ← 分解
    ...    ...    ...
   基础解  基础解  基础解     ← 求解（递归到底）
        \    |    /
        原问题的解          ← 合并
```

## 适用条件（四大特性）

| # | 特性 | 说明 |
|:-:|------|------|
| ① | 规模缩小到一定程度可解 | 递归终止条件 |
| ② | 可分解为若干子问题 | 子问题性质相同，规模更小 |
| ③ | 子问题的解可合并为原问题的解 | 合并步骤是关键 |
| ④ | **子问题互相独立** | 与动态规划的根本区别 |

## 经典应用框架

### 1. 二分搜索 — O(logn)

```
分解：取中间元素，与目标比较
求解：在左半或右半继续搜索
合并：无（直接返回结果）
```

### 2. 快速排序 — O(nlogn)

```
分解：选枢轴，快排分割将数组分为左小右大
求解：递归排序左右两部分
合并：无（原地排序）
```

### 3. 归并排序 — O(nlogn)

```
分解：从中间将数组分为两半
求解：递归排序左右两部分
合并：合并两个有序数组（需要额外空间 O(n)）
```

### 4. 快排分割求 Top K

利用快排的 partition 函数，每次确定一个元素的最终位置，判断该位置与 K 的关系，只递归包含第 K 大的那一侧。
